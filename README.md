# Product 6 — Liquor Store POS (UI-first phase)

This is the **Sale / Billing screen only**, built to match your reference
design 1:1 — feature-first architecture, reusable design-system
components, Riverpod, Go Router, Google Fonts, Material 3. No backend,
no Drift yet — the item table and totals are hard-coded dummy data on
purpose, per your "UI first" request.

## Why there's no `windows/` or `macos/` folder in this zip

Those native runner folders are generated by the Flutter SDK itself
(`flutter create .`) and are large/binary-ish + machine-specific. Shipping
ones I hand-wrote would be more likely to break than help. Two commands
below regenerate them correctly on **your** machine in about 10 seconds.

## Run it

```bash
# 1. unzip, then cd into the folder
cd pos_app

# 2. generate the native desktop runners for your OS
flutter create --platforms=windows,macos,linux .

# 3. get packages
flutter pub get

# 4. run (pick the one for your OS)
flutter run -d windows
flutter run -d macos
flutter run -d linux
```

Requires Flutter 3.19+ with desktop support enabled
(`flutter config --enable-windows-desktop` /
`--enable-macos-desktop` / `--enable-linux-desktop` if you haven't
already turned those on).

## What's in here

```
lib/
  app/                    → MaterialApp.router + GoRouter config
  core/theme/             → colors, spacing/radius, typography, ThemeData
                            (edit these 4 files to re-theme the whole app)
  shared/widgets/
    buttons/              → AppButton + PrimaryButton/SecondaryButton/
                            SuccessButton/DangerButton wrappers
    inputs/                → AppTextField, SearchField
    badges/                → KeyboardShortcutBadge, StatusChip
    layout/                → AppShell, AppSidebar, AppTopHeader,
                            AppStatusBar, AppCard, SectionHeader
  features/sales/
    domain/                → SaleLineItem model (dummy data for now)
    presentation/
      screens/             → SalesBillingScreen
      widgets/             → InvoiceDetailsCard, ScanAddItemCard,
                            ItemDetailsTable, BillSummaryCard,
                            PaymentCard, BillingModeToggle
      sales_providers.dart → Riverpod state (billing tab, payment method)
```

Every card on the Sale screen (Invoice Details, Scan/Add Item, Item
Details, Bill Summary, Payment) is its own file under 150 lines, built
on the shared `AppCard` + `SectionHeader` + button/field components —
so the same pieces will reuse cleanly for Purchase Bill, Products, etc.

## Working right now

- Cash / Card / UPI payment method selector — actually switches state
